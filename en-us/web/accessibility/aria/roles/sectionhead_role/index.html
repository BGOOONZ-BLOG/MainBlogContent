The sectionhead role, an abstract role, is superclass role for labels or summaries of the topic of its related section.

Note: The sectionhead role is an abstract role. It is included here for completeness of documentation. It should not be used by web authors.

Description

The structural sectionhead role is an abstract role for the subclass roles that identify the labels or summaries of the sections they label. The role must not be used. The four subclasses – columnheader, heading, rowheader, and tab. There are HTML element equivalents, like, {{HTMLElement(‘th’, ‘<th scope=“col”>’)}} for columnheader, {{HTMLElement(‘th’, ‘<th scope=“row”>’)}} for rowheader, and any of the HTML headings, {{HTMLElement(‘h1’)}} thru {{HTMLElement(‘h6’)}} for heading. The tab role does not currently have an HTML equivalent.

Best Practices

Do not use.
