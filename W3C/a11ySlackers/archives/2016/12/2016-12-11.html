    room for fat fingers
  3. that interactives are interactive in the source order
  4. (like if you move from focusable to focusable it goes in the expected visual order)
  5. that hidden things are both hidden and not interactable
  6. Even though it's developers that need to implement that stuff, Designers need to have it in mind while they design.
  7. Designers also should think about "how does my design work when X is open" where X is some interactive part of the phone like the onscreen keyboard.
  8. I've tried using websites where the keyboard always got in the way because once I focussed on something the page shifted wrong
  9. With the fat fingers thing, not only does the interactive have to be large enough, but also the space between interactives, and also they should consider if there will be something visible around the finger as the user taps it (like sometimes keyboards will have the key name stick out briefly while you tap it)
